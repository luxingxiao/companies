Qt Suite is a complete Qt development and testing toolkit. Domain skills load contextually, specialist agents activate proactively during development and after code changes. Six commands cover project scaffolding, component generation, test generation, test execution, coverage analysis, and headless visual verification.

## Agents

- **qt-app-dev** — Proactive Qt development specialist, triggers when creating new Qt projects or implementing widgets
- **qt-debugger** — Proactive diagnostics for Qt errors, crashes, frozen UIs, or unexpected widget behavior
- **qt-app-reviewer** — Proactive code quality reviewer, checks for GC risks, threading violations, missing object names
- **qt-ux-advisor** — Proactive UI/UX reviewer, enforces keyboard navigation, accessibility, and widget naming for testability
- **test-generator** — Coverage-gap-driven test generation, scans source for untested paths and writes targeted test files
- **gui-tester** — Autonomous visual testing via Qt Pilot, launches app headlessly via Xvfb and interacts by objectName

## Skills

- **qt-architecture** — QApplication setup, project layout, structuring a Qt app
- **qt-signals-slots** — Custom signals, cross-thread communication, signal/slot connections
- **qt-layouts** — Widget arrangement, resize behavior, QSplitter, layout debugging
- **qt-model-view** — QAbstractTableModel, QTableView, QSortFilterProxyModel, delegates
- **qt-threading** — QThread, QRunnable, thread safety, keeping UI responsive
- **qt-styling** — QSS stylesheets, theming, dark/light mode, QPalette
- **qt-resources** — .qrc files, pyrcc6, embedding icons and assets
- **qt-dialogs** — QDialog, QMessageBox, QFileDialog, custom dialogs
- **qt-packaging** — PyInstaller, Briefcase, platform deployment, CI builds
- **qt-debugging** — Qt crashes, widget visibility, event loop, threading issues
- **qt-qml** — QML/Qt Quick, QQmlApplicationEngine, exposing Python to QML
- **qt-settings** — QSettings, persistent preferences, window geometry, recent files
- **qt-bindings** — PySide6 vs PyQt6 differences, PyQt5 migration guide
- **qtest-patterns** — QTest (C++), pytest-qt (Python), QML TestCase test patterns
- **qt-coverage-workflow** — Coverage gaps, gcov, lcov, coverage.py workflows
- **qt-pilot-usage** — Headless GUI testing, widget interaction, Qt Pilot MCP usage
